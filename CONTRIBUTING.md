# 🤝 IT-Certifications-Roadmap Katkıda Bulunma Kılavuzu

Bu repository, Bilgi Teknolojileri sektöründeki uygulamalı ve endüstri standardı haline gelmiş sertifikaların tarafsız bir derlemesidir.

Bu rehbere yapacağınız katkılarda aşağıdaki kurallara uyulması gerekmektedir. 

## 📌 Genel Beklentiler
1. **İçerik Uygunluğu:** Eklenmesi planlanan sertifikanın veya konunun endüstri tarafından tanınan, spesifik donanım veya yazılım bilgisi barındıran yetkinlik sertifikalarından olması beklenmektedir.
2. **Uygulama Odaklılık:** Eklenen pratik yöntemlerinin veya çalışma platformlarının (lab, simülatör, kaynak doküman v.b.) somut olmasını rica ederiz.
3. **Sade Dil:** Açıklamaların tamamen tarafsız, kişisel görüşlerden uzak ve profesyonel bir tonda olması gerekmektedir.

## 🛠️ Nasıl Katkı Sağlanır?

### 1. Yeni Bir İçerik Önermek
- Repoya doğrudan katkıda bulunmadan önce `.github/ISSUE_TEMPLATE/feature_request.md` şablonunu doldurarak bir Issue açarak konuyu tartışmaya açabilirsiniz.

### 2. Geliştirme Süreci
1. Repoyu Fork'layın.
2. Yeni bir branch oluşturun (Örn: `feature/CyberSecurity-eJPT-Eklemesi`).
3. İlgili ana dizin altındaki belgelere uygun formatta ekleme yapın.
4. Eklemeniz şu formatta olmalıdır:
   - **Nedir:** (Kısa, net tanım)
   - **Kime Hitap Eder:** (Hangi rollere uygun olduğu)
   - **Sınav Formatı:** (Süre, formasyon)
   - **Hazırlık Yöntemi:** (Geçerliliği kabul görmüş eğitim platformları veya lab yaklaşımları)

### 3. Pull Request (PR) Açma
1. Değişikliklerinizi açık commit mesajlarıyla kaydedin:
   ```bash
   git commit -m "docs(cybersecurity): CompTIA Security+ sertifikası eklendi"
   ```
2. Fork ettiğiniz repoya `push`layın.
3. PR oluştururken mevcut `PULL_REQUEST_TEMPLATE.md` yönergelerine dikkat edin.

## ⌨️ Format Kuralları
- Karakter kodlaması **UTF-8** olmalıdır.
- Markdown başlık yapısı mevcut standartta korunmalıdır.
